# RouteMind 智行策 产品评估

RouteMind 智行策是面向本地生活路线规划的智能产品。它把自然语言目标、POI 质量、口碑热度、营业状态、移动成本和用户场景结合起来，生成可执行路线或候选推荐。

## 产品价值

传统 POI 搜索通常只回答“附近有什么”。RouteMind 智行策更进一步回答：

- 用户到底想找一家店，还是想安排一条路线。
- 哪些地点适合当前时间和服务范围。
- 多个地点之间是否能顺路到达。
- 为什么这些地点值得推荐。
- 在目标不清楚时应该追问什么。

## 能力完整性

| 能力 | 评估 |
|------|------|
| 自然语言理解 | 支持餐饮、购物、咖啡、夜间活动、商务短程、半日路线等常见表达 |
| 推荐排序 | 综合质量、热度、品牌、营业状态、距离和实体可信度 |
| 路线规划 | 输出时间轴、移动距离、移动时间和地图折线 |
| 多轮交互 | 支持沿用上次位置和偏好 |
| 澄清能力 | 对模糊短句返回可点击方向 |
| 服务边界 | 明确当前覆盖成都武侯区与锦江区 |
| 隐私设计 | 不收集真实身份、消费记录或长期位置轨迹 |

## 差异化

### 从搜索到决策

RouteMind 智行策不只是列出附近 POI，而是把用户目标转成可执行方案。比如“太古里附近逛街喝咖啡”会被理解为“商场 -> 咖啡/饮品”的组合，而不是只返回咖啡店列表。

### 从评分到综合质量

推荐排序不只看单一评分，还会结合评价热度、品牌识别、商圈密度、营业时间、移动成本和门店实体特征，降低误分类和低价值点位。

### 从单轮到连续对话

同一会话下，用户可以继续说“那附近换成茶馆”“还有别的吗”。系统会沿用上下文，减少重复输入。

## 当前服务范围

当前服务范围为成都武侯区与锦江区。系统不会对暂未覆盖区域输出缺少数据支持的推荐。扩展更多城市时，可以按城市接入对应 POI、评分口碑、营业时间和路径服务。

## 质量保障

- 明确过滤停业、装修、入口、门岗、停车、快递、充电等低价值点位。
- 对火锅、茶馆、咖啡、公园、商场、超市、酒吧等高频类型做实体匹配。
- 对组合需求保持顺序约束。
- 对不明确目标先澄清，不强行生成低确定性路线。
- 对暂未覆盖区域给出清晰服务范围提示。

## 扩展方向

- 接入更多城市数据。
- 接入线上评分、热度和营业时间服务。
- 接入更完整的地图路径服务。
- 增强实时排队、天气和人流等聚合信号。
- 增强用户授权下的偏好管理。
