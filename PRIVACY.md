# RouteMind 智行策 隐私设计白皮书

RouteMind 智行策的隐私原则是：路线体验的提升不以收集敏感个人信息为代价。系统只使用路线规划所需的信息，并提供会话和轻量画像的清除能力。

## 1. 不收集的信息

RouteMind 智行策不收集、不存储、不使用以下信息：

| 类别 | 示例 |
|------|------|
| 个人身份 | 手机号、姓名、身份证号、真实账号 |
| 长期轨迹 | 长期位置轨迹、跨会话精确行动记录 |
| 消费信息 | 支付记录、消费金额、银行卡信息 |
| 社交关系 | 好友列表、同行人关系、社交账号绑定 |
| 生物特征 | 人脸、声纹、步态 |

## 2. 可用信息

| 信息 | 用途 | 隐私控制 |
|------|------|----------|
| 本次输入 | 理解目标并生成路线 | 仅用于当前请求和会话 |
| 会话上下文 | 支持“那附近”“换一个”等连续表达 | 可主动清除 |
| 当前位置 | 计算附近 POI 和移动距离 | 需用户授权，不保存长期轨迹 |
| 当前时间 | 判断早餐、午餐、夜间等场景 | 公共上下文 |
| 天气和人流等聚合信息 | 优化室内外和排队体验 | 仅使用匿名聚合信号 |
| 轻量画像 | 保存偏好、避让、饮食限制摘要 | 可通过接口删除 |

## 3. 会话记忆

会话记忆用于同一轮对话中的连续规划。例如：

- 用户说“不要火锅”，后续同一会话会避开火锅。
- 用户说“那附近换成茶馆”，系统会沿用上次位置。
- 用户说“还有别的吗”，系统会避免重复刚推荐过的 POI。

会话可通过 `/api/session/clear` 清除。

## 4. 轻量画像

轻量画像仅保存低敏摘要：

- 偏好类型。
- 避让类型。
- 饮食限制。
- 常见区域摘要。

不保存真实身份、消费记录和完整位置轨迹。用户可通过 `/api/profile/clear` 删除指定画像。

## 5. 数据最小化

| 数据 | 保存方式 |
|------|----------|
| GPS 坐标 | 仅用于当前规划 |
| 会话上下文 | 用于当前会话，可主动清除 |
| 轻量画像 | 用户授权标识下保存摘要，可删除 |
| 路线结果 | 返回给用户，不作为长期行为轨迹 |

## 6. 用户控制

- 清除本次会话。
- 清除长期轻画像。
- 不使用当前位置，改用手动选择区域。
- 不提供用户标识，仅使用当前请求生成结果。

## 7. 合规设计

RouteMind 智行策遵循最小必要原则：

- 只收集路线规划所需的信息。
- 精确位置需要用户授权。
- 用户可清除会话和轻量画像。
- 不把路线规划数据用于无关用途。
- 第三方数据接入需遵守对应授权范围和数据使用条款。
