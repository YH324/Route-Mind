# 隐私设计白皮书

> **核心原则**：用户体验的提升绝不以牺牲用户隐私为代价。比赛版只使用会话状态、公开上下文和可删除的低敏轻画像，不收集手机号、真实身份、消费记录或长期位置轨迹。

---

## 1. 隐私边界定义

### 1.1 绝对禁区（不收集、不存储、不使用）

| 类别 | 具体数据 | 原因 |
|------|---------|------|
| **个人身份** | 手机号、姓名、身份证号、设备唯一标识符（IMEI/IDFA） | 《个人信息保护法》明确定义为敏感个人信息 |
| **真实身份跨会话追踪** | 手机号/账号绑定、历史搜索明细、长期位置轨迹 | 形成可识别个人画像，可追踪个人行为模式 |
| **金融信息** | 消费记录、支付金额、银行卡信息 | 超出路线规划业务边界，合规风险极高 |
| **社交关系** | 好友列表、同行人信息、社交账号绑定 | 与核心功能无关，且易泄露社交图谱 |
| **生物特征** | 人脸、声纹、步态 | 与路线规划完全无关 |

### 1.2 可用信息池（不关联个人身份）

| 类别 | 具体数据 | 使用方式 | 匿名性保障 |
|------|---------|---------|-----------|
| **会话内交互** | 本次对话中的query、点击、反馈 | 仅用于同一会话内的路线调整 | 会话结束后立即清除，不持久化 |
| **当前位置** | GPS坐标（用户主动授权） | 计算POI距离、确定中心点 | 仅用于本次规划，不存储历史轨迹 |
| **当前时间** | 小时、星期、日期、季节 | 时间感知路由（早餐/下午茶/夜宵） | 公开信息，无需授权 |
| **天气信息** | 温度、降雨、空气质量 | 天气感知推荐（雨天推荐室内） | 调用公开天气API，不关联用户 |
| **POI聚合统计** | 某POI的实时排队人数、时段人流密度、近期评分趋势 | 实时人流规避、热门度排序 | 匿名统计，无法反推个人 |
| **匿名群体偏好** | "周六晚上春熙路的人68%选择火锅" | 推荐时作为群体信号加权 | 聚合数据，N≥100 |
| **设备类型** | 手机/平板（粗略判断） | 移动端优化显示 | 不收集具体型号，不关联身份 |
| **粗略地理** | 城市级别（如"成都市"） | 加载对应城市数据分片 | 不存储区县级以下位置 |
| **长期轻画像** | `preferred_tags`、`avoid_tags`、`dietary`、常见区域摘要 | 仅在用户提供 `user_id` 时用于偏好继承 | 不保存真实身份，不保存完整轨迹，可通过 `/api/profile/clear` 删除 |

### 1.3 灰色地带（可用但需强化告知）

| 数据 | 使用场景 | 必要措施 |
|------|---------|---------|
| 精确GPS坐标 | 计算附近POI距离 | 必须弹窗授权，可随时撤回；提供"手动输入位置"替代方案 |
| 相册/相机 | 用户主动上传打卡照片 | 仅用于当前展示，不上传云端，不分析EXIF位置 |

## 2. 轻个性化算法设计

### 2.1 会话内上下文记忆（Session Context）

**设计原则**：会话记忆只存在于当前服务进程内，按 TTL 自动过期，可通过 `/api/session/clear` 主动清除；长期偏好不写入会话日志，只进入可删除的轻画像文件。

```python
class SessionContext:
    """会话级上下文，会话结束即销毁"""
    def __init__(self):
        self.queries = []           # 本次会话的所有query
        self.excluded_pois = set()  # 用户明确排除的POI
        self.excluded_types = set() # 用户明确排除的类型（如"不要火锅"）
        self.clicked_pois = set()   # 用户点击/展开详情的POI
        self.preferred_types = []   # 用户多次提及的偏好类型
        self.radius_adjustment = 0  # 用户反馈"太远/太近"后的半径调整
        self.last_route_end = None  # 上一条路线的终点（用于连贯性）
```

**使用场景**：

| 用户行为 | 上下文更新 | 后续影响 |
|---------|-----------|---------|
| "不要火锅" | `excluded_types.add("火锅")` | 后续规划自动排除所有火锅 |
| "太远了" | `radius_adjustment -= 500` | 缩小搜索半径，推荐更近的POI |
| 点击"无二泰式火锅" | `clicked_pois.add("B0xxxxx")` | 同类型POI权重+0.5 |
| "再推荐一条" | `last_route_end = 路线终点` | 新路线从终点开始，不重复 |
| "不要刚才那些" | `excluded_pois.update(已推荐POI)` | 全新候选集，排除已推荐 |

### 2.2 时间感知路由（Time-Aware Routing）

**信息来源**：系统当前时间（公开信息）
**隐私风险**：零

```python

    "dinner":     {"hours": (18, 21), "boost": ["火锅", "烧烤", "中餐", "外国菜"],         "avoid": ["早餐"]},
    "night":      {"hours": (21, 24), "boost": ["烧烤", "酒吧", "夜宵", "KTV"],           "avoid": ["景点", "公园"]},
```

**效果示例**：
- 用户下午3点说"想吃点东西" → 优先推荐下午茶/甜品/咖啡，而非火锅
- 用户晚上9点说"想吃点东西" → 优先推荐烧烤/夜宵/酒吧
- 用户早上8点说"想逛公园" → 正常推荐，但推荐有晨练设施的公园

### 2.3 天气感知推荐（Weather-Aware Recommendation）

**信息来源**：公开天气API（如和风天气、OpenWeatherMap）
**隐私风险**：零

```python
WEATHER_RULES = {

    "snow":       {"boost": ["火锅", "温泉", "商场"],                   "avoid": ["公园", "景点"], "message": "雪天推荐温暖的室内"},
```

- 用户说"想逛公园"，但AQI>150 → 提示空气质量，推荐室内植物园或商场顶层花园

### 2.4 实时人流规避（Crowd Avoidance）

**信息来源**：POI层面的匿名聚合数据（如美团排队人数、高德人流指数）
**隐私风险**：匿名统计，无法反推个人

```python
def apply_crowd_penalty(poi, current_time, crowd_data):
    """
    高峰时段对排队严重的POI降权
    crowd_data: {poi_id: {"wait_min": 30, "crowd_level": "high"}}
    """
    crowd = crowd_data.get(poi["poi_id"])
    if not crowd:
        return 0
    
    wait = crowd.get("wait_min", 0)
    if wait > 60:
        return -3.0  # 严重惩罚
    elif wait > 30:
        return -1.5
    elif wait > 15:
        return -0.5
    return 0
```

**效果示例**：
- 周六晚上6点推荐火锅 → 楠火锅排队120分钟（惩罚-3分），自动推荐附近排队短的替代店
- 用户说"不想排队" → 额外排除所有 `wait_min > 30` 的POI

### 2.5 路线连贯性（Route Continuity）

**设计原则**：同一会话内的多条路线之间保持空间和时间连贯性。

| 场景 | 连贯性策略 |
|------|-----------|
| 用户说"再推荐一条" | 新路线从上次终点开始，排除已去过的POI |
| 用户说"吃完去哪儿" | 以上一顿餐厅为起点，推荐附近休闲/娱乐 |
| 用户说"明天继续" | 会话已断，不继承任何上下文（保护隐私） |
| 用户调整时间 | 保持POI顺序不变，仅调整停留时间 |

### 2.6 匿名群体信号（Aggregate Signals）

**设计原则**：利用大规模匿名统计优化推荐，但绝不形成个人画像。

```python
AGGREGATE_SIGNALS = {
    "weekend_evening_chunxi": {"火锅": 0.68, "烧烤": 0.45, "酒吧": 0.32},
    "rainy_day_indoor": {"商场": 0.55, "电影院": 0.38, "茶馆": 0.25},
    "holiday_tourist": {"景点": 0.72, "小吃": 0.58, "纪念品": 0.30},
}
```

**使用方式**：
- 不是"因为你喜欢火锅所以推荐火锅"
- 而是"周六晚上春熙路68%的人选择火锅，这是群体偏好信号"
- 信号仅作为权重微调（±0.3分），不决定最终结果

**数据来源**：
- 美团/点评的时段销量统计（匿名聚合）
- 高德人流热力图（匿名聚合）
- 小红书POI打卡时段分布（匿名聚合）

## 3. 技术保障措施

### 3.1 会话隔离

```python
# 会话数据存储在内存，Redis设置TTL=会话超时时间
session_data = {
    "session_id": "anon_uuid_123",
    "created_at": "2024-03-15T16:00:00Z",
    "context": SessionContext(),
    "ttl_seconds": 1800,  # 30分钟无操作自动销毁
}
```

### 3.2 数据最小化

| 数据 | 存储时长 | 存储位置 | 是否加密 |
|------|---------|---------|---------|
| 会话上下文 | 会话活跃期间 | 内存/Redis | 否（已是匿名UUID） |
| 长期轻画像 | 用户主动清除前 | 本地 `output/user_memory_profiles.json` / 未来画像服务 | 比赛版本地文件，生产应加密 |
| GPS坐标 | 仅本次规划 | 内存 | 否（不持久化） |
| 查询日志 | 7天（脱敏后） | 日志系统 | 是（AES-256） |
| 路线结果 | 不存储 | 仅返回给用户 | N/A |

### 3.3 用户控制

- **"清除本次会话"**：一键销毁所有会话内上下文
- **"清除长期画像"**：删除指定 `user_id` 的轻量偏好、避让和饮食限制
- **"不使用我的位置"**：提供手动输入位置替代GPS
- **"关闭个性化"**：关闭时间感知、天气感知、群体信号，仅保留基础规则

## 4. 合规对照

| 法规 | 要求 | 本项目措施 |
|------|------|-----------|
| **《个人信息保护法》** | 最小必要原则 | 只收集路线规划必需的信息（位置、时间） |
| **《个人信息保护法》** | 用户同意 | GPS精确位置弹窗授权，可随时撤回 |
| **《个人信息保护法》** | 目的限制 | 所有数据仅用于路线规划，不做他用 |
| **《数据安全法》** | 数据分类分级 | 会话数据为"低敏感"，不存储任何个人身份信息 |
| **GDPR（如出海）** | 被遗忘权 | 会话可手动清除，长期轻画像可通过 `/api/profile/clear` 删除 |
| **GDPR** | 数据可携带 | 当前仅保存低敏偏好摘要，生产化时再提供导出接口 |

---

## 5. 与三种用户模式的结合

| 用户模式 | 可用的轻个性化 | 不可用的（保护匿名性） |
|---------|-------------|---------------------|
| **游客** | 时间感知、天气感知、群体信号、实时人流、可删除轻画像 | 真实身份历史、消费记录 |
| **出差** | 时间感知（碎片化时间适配）、实时人流（效率优先） | 公司信息、行程信息 |
| **居民** | 时间感知、天气感知、实时人流、新店发现 | 家庭住址、日常轨迹 |

## 6. 未来扩展（需额外授权）

| 功能 | 需要的数据 | 授权方式 |
|------|-----------|---------|
| **账号级长期画像** | 真实账号体系 + 授权偏好 | "记住我的偏好"开关 + 删除/导出入口 |
