# RouteMind 架构说明

本文档说明 RouteMind 的系统架构、核心模块和扩展设计。

## 1. 总体架构

```text
用户输入
  -> Web 工作台
  -> 应用服务层
  -> 交互理解层
  -> 路线规划与排序引擎
  -> 数据适配层
  -> POI / 评分口碑 / 营业时间 / 路径 / 语义向量
```

RouteMind 的核心设计是规划引擎与数据来源解耦。无论数据来自本地资产还是线上服务，只要适配层返回统一字段，规划引擎都可以保持稳定。

## 2. 模块清单

| 模块 | 文件 | 说明 |
|------|------|------|
| Web 工作台 | `web/index.html`、`web/app.js`、`web/styles.css` | 自然语言输入、地图、路线、推荐理由展示 |
| 应用服务 | `app_service.py`、`web_app.py` | 请求处理、服务范围判断、结果包装 |
| 交互理解 | `interaction_intelligence.py` | 多轮上下文、多人对话、偏好和避让 |
| 路线规划 | `route_planner_v3.py` | 意图理解、候选筛选、排序和路线构建 |
| 路网计算 | `road_network.py`、`poi_knn_graph.py` | 距离、移动时间和路线折线 |
| 语义检索 | `semantic_search.py` | 复杂目标的语义候选增强 |
| 类型与评分 | `ugc_type_profiles.py` | 类型识别、实体治理和评分特征 |
| 配置 | `config.py` | 服务配置和密钥读取 |

## 3. 数据适配层

数据适配层统一输出以下能力：

- POI 搜索。
- POI 详情。
- 评分与口碑摘要。
- 营业时间。
- 路径距离、时间和折线。

因此，接入线上数据服务时不需要重写规划引擎，只需要让适配层保持字段契约一致。

## 4. 路线规划引擎

规划引擎负责：

- 解析用户目标。
- 判断单点、组合路线或完整路线。
- 估算时间预算。
- 过滤服务范围和低价值候选。
- 结合评分、热度、营业时间、距离、实体可信度和语义需求排序。
- 构建时间轴路线。
- 输出结构化推荐依据。

## 5. 推荐依据

每个推荐点会返回 `recommendation_basis`，包含：

- 质量评分。
- 类型权重。
- 用户偏好命中。
- 评价热度估计。
- 品牌和实体可信度。
- 营业状态。
- 起点距离和移动成本。
- 路线顺序匹配。

前端会把这些字段转成用户可读的推荐理由。

## 6. 用户模式

| 模式 | 目标 | 主要约束 |
|------|------|----------|
| 游客 | 丰富体验和城市特色 | 更大半径、景点和特色餐饮优先 |
| 出差 | 时间效率 | 更短半径、更短移动时间、商务餐饮优先 |
| 居民 | 日常便利 | 周边品质、茶馆餐饮、公园和社区便利优先 |

## 7. 服务范围

当前服务范围为成都武侯区与锦江区。服务范围由应用层统一判断，前端和接口响应保持一致。暂未覆盖地区会返回明确提示。

## 8. 城市扩展

扩展更多城市时，每个城市维护独立资产：

- POI 基础信息。
- 评分与口碑索引。
- 类型索引和空间索引。
- 营业时间。
- 路网或路径缓存。
- 语义向量。

服务可按城市加载资产，并在热门城市预热缓存。

## 9. 隐私设计

RouteMind 不依赖敏感个人信息。会话上下文只服务于连续对话，轻量画像只保存偏好、避让和饮食限制摘要，并支持主动清除。详细说明见 `PRIVACY.md`。

## 10. 运行

```bash
python run_server.py --warmup
```

访问：

```text
http://127.0.0.1:8000/
```

健康检查：

```text
http://127.0.0.1:8000/api/health
http://127.0.0.1:8000/api/ready
```
