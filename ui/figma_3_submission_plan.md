# Figma 三次提交计划

## 第 1 次：主界面视觉确认

目标：锁定整体风格，不先做复杂状态。

提交内容：

- Desktop 1440 x 900 主工作台。
- 插画地图底图：道路、楼宇、公园、树木、花草。
- POI 类型图标：景点、餐饮、茶馆、夜逛、起点。
- 左侧输入、新手指引、用户模式、皮肤选择。
- 右侧方案卡、时间轴、推荐理由。

需要 Figma 重点做：

- 地图插画质感精修。
- 图标风格统一，避免 emoji 的临时感。
- 整体颜色更贴近本地生活产品：温暖、可信、易扫描。

## 第 2 次：组件与交互状态

目标：把主界面拆成可落地组件。

提交内容：

- 语言切换组件。
- 用户模式 segmented control。
- 皮肤选择组件。
- 地图工具按钮：放大、缩小、归中、图层。
- 方案卡 active/inactive。
- 时间轴 stop collapsed/expanded。
- Toast、Tooltip、新手指引卡。

需要 Figma 重点做：

- 按钮 hover/focus/disabled 状态。
- 地图 tooltip 展开态。
- 时间轴节点展开态。
- 新手指引跳过后的布局状态。

## 第 3 次：完整流程与边界状态

目标：形成可用于评审和开发对齐的 prototype。

提交内容：

- 方案切换 prototype。
- 皮肤切换 prototype。
- 复制路线 toast。
- 生成中状态。
- 无结果修正状态。
- 单点推荐状态。

需要 Figma 重点做：

- Prototype 连线。
- 核心使用路径标注。
- 无结果时的三个修正按钮：扩大半径、改时间、切换模式。
- 导出给前端的标注：色板、字体、间距、组件命名。
