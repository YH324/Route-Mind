# 默认值与语义兜底说明

RouteMind 不维护预先写死的推荐门店列表，也不输出固定路线模板。运行时推荐由本地 POI 语料、质量信号、评价热度估计、语义标签、营业时间、路网距离、服务范围、用户模式和会话上下文共同生成。

## 必要默认值

- **服务范围**：当前产品范围是成都武侯区和锦江区。超出范围的请求会在规划前被拒绝，避免系统假装拥有未覆盖地区的数据。
- **中心坐标**：用户没有提供位置时，API 使用成都中心点作为安全起点；如果文本中提到已支持地标，请求会重定位到该地标。
- **搜索半径**：缺失或非法半径会被归一到有边界的运行半径，防止后端执行无上限扫描。
- **时间预算**：规划器会按意图形态和活动组合推断未明说的时长。历史 `TIME_BUDGET=4` 环境变量仅保留兼容，不再作为所有请求的通用四小时方案。

## 确定性语义兜底

后端保留确定性意图门控，用于识别“附近火锅”和“半日路线”这类高置信差异。配置了大模型服务时，系统会先尝试 MiMo / MiniMax / GLM 做意图判断，再用确定性门控修正明显误判；未配置或服务不可用时，本地语义兜底会保持服务可用，并继续基于数据对候选 POI 排序。外部模型失败后会进入短时熔断，避免每个请求都反复等待失败接口。它不是推荐模板，也不会写死门店或路线。

## 已清理的产品可见痕迹

- 模糊追问选项会根据当前请求中心生成，不再固定给春熙路 / 武侯祠 / 太古里示例。
- 服务范围错误里的快捷动作会根据用户当前选择的支持中心生成。
- 公开模型元数据报告 `deterministic_gate` 或 `local_semantic_guardrail`，便于观测意图来源，同时避免让人误解为硬编码门店推荐。
