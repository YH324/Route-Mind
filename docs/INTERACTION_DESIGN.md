# RouteMind 智行策 交互能力设计

RouteMind 智行策的交互目标是让用户以自然语言连续表达本地生活需求，并在目标不明确时获得清晰引导。

## 1. 会话记忆

会话记忆用于支持同一轮对话中的连续规划。

示例：

- 用户先说“春熙路附近吃火锅”，再说“那附近有什么商场”，系统沿用春熙路作为中心。
- 用户说“不要烧烤”，后续同一会话会避开烧烤。
- 用户说“还有别的吗”，系统避免重复刚推荐过的 POI。

会话记忆可通过接口清除。

## 2. 轻量偏好

在用户提供授权标识时，系统可以保存低敏偏好摘要：

- 偏好类型。
- 避让类型。
- 饮食限制。
- 常见区域摘要。

这些信息不包含真实身份、消费记录或长期位置轨迹，并支持删除。

## 3. 多人对话

RouteMind 智行策支持多人讨论式输入，例如：

```text
小明：春熙路附近吃火锅
小红：吃完想逛街
小明：不要太贵
```

系统会聚合地点、餐饮、活动、预算和顺序需求，生成更符合多人共同目标的路线。

## 4. 语义需求

系统会把自然语言中的隐含需求转成可排序信号：

| 表达 | 推荐倾向 |
|------|----------|
| 清淡、不辣 | 粤菜、菌汤锅、轻食、茶馆 |
| 约会、有氛围 | 安静、环境好、适合聊天 |
| 带孩子 | 公园、亲子餐厅、空间更宽松的地点 |
| 商务 | 中餐、茶馆、短距离和稳定营业 |
| 拍照好看 | 景点、公园、商圈和环境较好的店 |

## 5. 澄清问题

当用户只说“随便安排一下”“附近有什么”等目标时，系统会先给出可点击方向：

- 餐饮。
- 景点/公园。
- 咖啡/茶馆。
- 夜间活动。
- 出差短程。

澄清选项会根据当前中心动态生成。

## 6. 用户控制

- 清除当前会话。
- 清除轻量偏好画像。
- 不使用当前位置，改用手动选择中心。
- 修改目标后重新规划。
