# RouteMind 智行策 上线准备清单

## 基础设施

- [ ] 服务器资源满足目标并发和数据加载需求。
- [ ] 域名、HTTPS、反向代理和静态资源缓存配置完成。
- [ ] 防火墙和访问控制策略配置完成。
- [ ] 健康检查和就绪检查接入部署平台。

## 数据服务

- [ ] POI 数据、评分口碑、营业时间和路网数据完成导入。
- [ ] 城市服务范围配置准确，页面和接口提示一致。
- [ ] 线上 POI、评分、营业时间和路径服务的授权与配额确认完成。
- [ ] 数据更新、备份和回滚策略确认完成。
- [ ] 城市级缓存策略确认完成。

## 推荐质量

- [ ] 每个路线点或推荐点都包含用户可读推荐理由。
- [ ] 火锅、茶馆、咖啡、公园、商场、超市、酒吧等高频类型完成质量验收。
- [ ] 组合需求保持用户表达顺序，例如商场到咖啡、酒吧到夜宵、午餐到咖啡。
- [ ] 未说明时长的请求会按目标类型估算合理预算。
- [ ] 服务范围外请求能清晰提示暂不支持。
- [ ] 严格时间或移动约束下不会硬凑低质量路线。
- [ ] 模糊短句能返回澄清选项并支持继续规划。

## 前端体验

- [ ] 首次进入页面展示服务范围公告，用户确认后不重复打断。
- [ ] 目标建议按当前中心动态生成。
- [ ] 地图能展示起点、POI、路线折线和详情弹窗。
- [ ] 右侧结果列表可滚动，多个推荐不会被遮挡。
- [ ] 推荐卡片和地图弹窗展示精选口碑摘要。
- [ ] 移动端规划、地图、结果三视图体验完整。
- [ ] 中英文和主题切换不影响核心流程。

## 安全与隐私

- [ ] API Key 使用密钥管理或环境变量保存，不写入代码仓库。
- [ ] 请求体大小、CORS、XSS 防护和访问日志策略确认完成。
- [ ] 用户位置只用于当前规划，不保存长期轨迹。
- [ ] 会话记忆和轻画像支持用户主动清除。
- [ ] 隐私政策和服务范围说明已对外展示。

## 运维

- [ ] `/api/health` 和 `/api/ready` 接入监控。
- [ ] 日志轮转、告警和故障恢复流程确认完成。
- [ ] 数据加载耗时、规划耗时、错误率和缓存命中率有可观测指标。
- [ ] 高峰并发下响应契约保持稳定。

## 合规

- [ ] 地图、POI、评分、评论和路径数据的授权范围确认完成。
- [ ] 第三方地图使用条款确认完成。
- [ ] 开源依赖协议完成核对。
- [ ] 用户隐私政策完成核对。
