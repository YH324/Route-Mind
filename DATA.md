## 2. POI本体数据

### 2.1 来源（预研阶段）
原始数据：高德地图 `gd_510000_poi.xlsx`（320万条，4sheet共1.5GB）
提取范围：**成都市武侯区+锦江区**
提取字段：`poi_id`, `name`, `type`, `address`, `longitude`, `latitude`, `typecode`, `cityname`, `adname`, `tags`, `tel`

### 2.2 未来来源（全国推广）
- **主数据源**：高德地图POI Search API / 百度地图Place API
- **辅助数据**：美团门店API（补充评分、评论数、营业时间）
- **加载策略**：用户query中解析城市 → 调用API获取该城市POI → 内存中构建空间索引和类型索引 → 使用完毕释放
- **缓存策略**：热门城市（北上广深蓉杭）POI数据本地缓存24h，冷门城市实时查询

### 2.5 已知数据质量问题
1. **Typecode映射错位**：部分POI的typecode与实际业务不符（如火锅店typecode标为"100700"实为"中餐"）
2. **Tag缺失/误导**：「童装」tag下混入大量「美甲」POI
3. **名称歧义**：「法国希思黎」被误识别为「外国菜」（实为化妆品品牌）
4. **坐标精度**：部分POI坐标精度约50-100m

## 3. Ground Truth评分体系

> **预研阶段**：规则生成GT评分，用于验证评分加权算法。
> **全国推广**：替换为平台真实评分（美团星级、高德评分、大众点评口味/环境/服务分）。

### 3.1 评价维度（预研）
每个POI有3-5个Ground Truth维度，按类型差异化：

| 维度类型 | 权重 | 说明 |
|---------|------|------|
| overall | 0.4-0.5 | 总体评价，所有POI必选 |
| tag_quality | 0.15-0.25 | 与标注tag匹配度 |
| 类型专属维度 | 0.2-0.4 | 各类型不同（如火锅→锅底/食材/辣度） |

### 3.2 未来真实评分映射
| 平台 | 字段 | 映射到本系统 |
|------|------|------------|
| 美团 | `avg_score` | overall |
| 大众点评 | `口味`/`环境`/`服务` | 类型专属维度 |
| 高德 | `biz_ext.rating` | overall辅助验证 |
| 小红书 | 笔记情感分析 | 口碑热度加成 |

### 3.2 部分类型维度配置
```python
{
  "火锅": {"overall": 0.4, "锅底": 0.2, "食材": 0.15, "辣度": 0.15, "服务": 0.1},
  "景点": {"overall": 0.4, "景观": 0.2, "人流": 0.15, "文化": 0.15, "设施": 0.1},
  "公园": {"overall": 0.4, "绿化": 0.2, "设施": 0.15, "步道": 0.15, "人流": 0.1},
  "餐饮-小吃": {"overall": 0.5, "味道": 0.2, "性价比": 0.15, "服务": 0.15}
}
```

### 3.3 评分方法
每条评论给各维度打1-5星，同POI所有评论取均值：
```python
# 最终GT评分 = weighted_sum(维度均值 × 维度权重)
# 范围: 1.0 - 5.0
```

## 4. UGC评论数据

> **预研阶段**：规则生成评论，验证类型差异化评论引擎。
> **全国推广**：接入平台真实评论，本章节生成规则仅保留为 fallback 方案。

### 4.1 预研阶段生成规模
- 总评论数：**7,754,232条**
- 覆盖POI：47,045个
- 平均评论/POI：**164.8条**
- 总大小：1.97GB

### 4.2 全国推广真实评论策略
| 平台 | 获取方式 | 量级 | 用途 |
|------|---------|------|------|
| 美团/点评 | 商家详情API | 单POI 10-1000条 | GT评分计算、关键词提取 |
| 小红书 | 搜索API+笔记内容 | 单POI 5-500条 | 网红热度、拍照打卡点 |
| 抖音 | POI视频数据 | 单POI 10-1000条 | 短视频热度、视觉吸引力 |

### 4.3 预研阶段类型差异化引擎（保留为fallback）
36种类型各有独立的：
- **评价维度**（火锅→锅底/食材/辣度/服务/等位）
- **词汇库**（火锅→"牛油""毛肚""九宫格"；公园→"绿化""步道""晨练"）
- **模板库**（不同星级×不同长度×不同类型的句子模板）
- **评论数量范围**（火锅8-18条→平均394条/POI；便利店2-6条→平均68条/POI）
- **长度分布权重**（火锅偏长篇15%，小吃偏短篇40%）

### 4.4 数量计算（预研）
```python
count = int(base_count × density_factor × quality_factor)
# density_factor: 网格密度0.5-2.0
# quality_factor: GT评分0.7-1.3
```

### 4.5 输出格式（紧凑JSON）
```json
{"poi_id":"B0FFGCTJKR","comments":[
  {"s":5,"d":{"overall":5.0,"锅底":4.8,"食材":5.0}},"t":"锅底香得遭不住...","l":128},
  ...

```

### 4.6 生成性能（预研）
- 流式输出，每2000 POI打印进度
- 使用 `json.dumps(..., separators=(',', ':'))` 最小化体积

## 5. 路网数据

> **预研阶段**：基于POI坐标模拟路网，验证Dijkstra算法和KNN缓存机制。
> **全国推广**：完全替换为高德/百度路径规划API，保留 `network.get_route_between()` 接口，内部实现切换即可。

### 5.1 预研阶段：模拟路网
从47,045个POI坐标构建图：
1. 500m网格空间索引，加速近邻查询
2. 每个POI连接k=10个最近邻居（距离<2000m）
3. 路段类型随机分配：次干道/小巷/步行街/主干道/快速路
4. 绕行系数：快速路1.0/步行街1.1/次干道1.2/主干道1.3/小巷1.5

### 5.2 模拟图规模
- 节点数：47,045
- 边数：291,460（双向）
- 平均节点度数：约6.2
- 平均绕行系数：**1.61**（实际/直线距离）

### 5.3 预研阶段：路径算法
- 单源Dijkstra + LRU缓存
- 冷缓存：首次从某起点出发约0.66秒
- 热缓存：后续查询约0.001秒
- 缓存命中：前200次规划几乎全热缓存

### 5.4 全国推广：真实路网API
| 能力 | 模拟路网 | 高德路径API |
|------|---------|------------|
| 精度 | 绕行系数1.61（估算） | 真实导航级精度 |
| 实时性 | 静态 | 实时路况（拥堵/封路） |
| 成本 | 零 | 日配额1万次，超出付费 |
| 覆盖 | 仅限POI点 | 全国道路网 |
| 响应 | 0.66s（冷）/0.001s（热） | ~150-300ms/次 |

**KNN缓存的价值**：API调用有成本和延迟，KNN懒加载缓存可以把常用POI-POI距离缓存到本地，热门区域规划零API调用。

## 7. 三种用户模式数据视图

### 7.1 游客模式
- 关注POI类型：景点、公园、火锅、小吃、茶馆、文化体验
- 距离容忍：5000m
- 时间窗口：09:00-22:00
- 营业校验：白天景点必须开放，晚餐火锅必须营业
- 排斥类型：便利店、超市、银行、医院、加油站

### 7.2 出差模式
- 关注POI类型：商务中餐、茶馆、精品简餐、按摩SPA
- 距离容忍：**1000m（硬约束）**
- 单段移动上限：**15分钟（硬约束）**
- 时间窗口：碎片化（午休12:00-14:00 / 晚19:00-22:00）
- 排斥类型：景点、游乐园、公园、购物（除商场）、便利店

### 7.3 居民模式
- 关注POI类型：火锅、烧烤、茶馆、公园、健身、超市
- 距离容忍：2500m
- 时间窗口：周末全天/工作日晚上
- 新店加成：开业<6个月POI评分+0.3
- 高峰规避：排队>4的POI在高峰时段降权
- 排斥类型：景点（游客区）、高消费场所
