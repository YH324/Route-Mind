# 数据资产说明

本文档说明 RouteMind 智行策当前使用的数据资产、字段结构和城市扩展方式。当前在线体验覆盖成都武侯区与锦江区，后续可按城市接入更多 POI、评分口碑、营业时间和路径服务。

| 数据文件 | 规模 | 用途 |
|---------|------|------|
| `wuhou_jinjiang_pois.json` | 47,045 POI | POI 基础信息、位置、类型、地址、标签 |
| `output/gt_index.json` | 47,045 条 | 评分与口碑聚合索引 |

## POI 基础信息

POI 数据包含：

- `poi_id`：唯一标识。
- `name`：名称。
- `type` / `typecode`：类型和类型编码。
- `address`：地址。
- `longitude` / `latitude`：坐标。
- `cityname` / `adname`：城市与区县。
- `tags`：业务标签。
- `grid_density`：商圈密度信号。
- `nearest_same_type_m`：同类型最近距离。

示例：

```json
{
  "poi_id": "poi-id",
  "name": "火锅门店",
  "type": "餐饮服务;中餐厅;火锅店",
  "address": "门店地址",
  "longitude": 104.08099,
  "latitude": 30.65732,
  "typecode": "050117",
  "cityname": "成都市",
  "adname": "锦江区",
  "tags": ["餐饮", "火锅"],
  "grid_density": 102,
  "nearest_same_type_m": 180
}
```

## 评分与口碑索引

评分与口碑索引用于生成推荐依据，包括：

- 总体质量评分。
- 类型相关评分维度。
- 评价热度估计。
- 适合场景标签。
- 用户可读的口碑摘要。

这些信号会和距离、营业时间、类型匹配、品牌识别和实体可信度一起参与排序。

## 类型索引

RouteMind 智行策支持餐饮、火锅、小吃、茶馆、饮品、景点、公园、商场、超市、酒吧、KTV、影院等常见本地生活类型。类型索引用于：

- 快速筛选候选 POI。
- 修正原始类型中可能出现的歧义。
- 保证组合路线中的类型顺序。
- 控制同类 POI 重复。

## 营业时间

营业时间数据用于判断预计到达时是否可用。字段包括：

```json
{
  "poi_id": "poi-id",
  "open": "10:30",
  "close": "02:00",
  "peak": "18:00-21:00",
  "rest_day": "无",
  "type": "火锅"
}
```

跨天营业、休息日和高峰时段会影响路线时间轴和排序。

## 路网数据

路网数据用于估算移动距离、移动时间和地图折线。路线构建会优先使用路网距离，而不是只用直线距离。

字段示例：

```json
{
  "node_id": "poi-id",
  "edges": [
    {
      "to": "next-poi",
      "distance_m": 234.5,
      "travel_time_min": 2.8,
      "road_type": "步行街"
    }
  ]
}
```

## 轻量画像数据

`output/user_memory_profiles.json` 只保存用户授权标识下的低敏摘要：

- 偏好类型。
- 避让类型。
- 饮食限制。
- 常见区域摘要。
- 更新时间。

不保存手机号、真实身份、消费记录或长期位置轨迹。用户可通过接口清除画像。

## 城市扩展策略

扩展更多城市时，每个城市独立维护：

- POI 基础信息。
- 评分与口碑索引。
- 类型索引。
- 空间索引。
- 营业时间。
- 路网或路径缓存。
- 语义向量。

服务可以按城市按需加载资产，热门城市可预热缓存。

## 数据质量治理

RouteMind 智行策会对候选 POI 做质量治理：

- 过滤停业、装修、入口、门岗、停车、快递、充电等低价值设施。
- 降低与用户意图不匹配的普通销售点、酒店房源、会展设施等候选。
- 结合名称、类型编码和标签修正茶馆、咖啡、公园、商场、酒吧等高频类型。
- 对火锅等强需求类型优先完整正餐门店。

## 对外展示原则

- 不编造数据资产中不存在的门店或景点。
- 不用服务范围外的数据替代当前请求城市。
- 推荐依据用用户可理解的质量、热度、距离、营业和场景信号表达。
- 当前服务范围、数据覆盖和扩展路径保持透明。
