# RouteMind 智行策 产品说明

RouteMind 智行策是一个面向本地生活场景的智能路线规划系统。用户用自然语言描述目标，系统自动识别需求、筛选 POI、生成时间轴路线或候选推荐，并给出清晰的推荐依据。

**项目试用**：[http://124.223.28.124/](http://124.223.28.124/)

## 产品定位

RouteMind 智行策解决的是“用户不知道如何把本地生活目标转成可执行路线”的问题。它不是普通 POI 搜索，也不是固定示例模板，而是把用户意图、地点质量、口碑热度、营业状态、移动成本和场景偏好组合成可执行方案。

当前在线体验以成都武侯区与锦江区作为样例数据区域；产品架构按多城市扩展设计，后续可接入更多城市的 POI、评分、营业时间和路径服务。

## 核心能力

| 能力 | 说明 |
|------|------|
| 自然语言理解 | 支持单点推荐、组合路线、半日/一日路线、商务短程、夜间活动等表达 |
| 动态时间预算 | 按目标类型估算合理用时，显式时长优先 |
| 路线生成 | 输出带时间轴、移动距离、移动时间和地图折线的路线 |
| 多方案对比 | 完整路线可生成不同风格方案，便于用户比较 |
| 推荐依据 | 每个推荐点展示质量、热度、类型命中、营业状态、移动成本和口碑摘要 |
| 多轮交互 | 支持“那附近”“换成茶馆”“还有别的吗”等连续表达 |
| 模糊澄清 | 对“随便安排一下”等不明确目标先询问方向 |
| 服务边界透明 | 暂未覆盖区域会明确提示，不返回缺少数据支持的推荐 |

## 典型场景

| 输入 | 输出倾向 |
|------|----------|
| 春熙路附近想吃火锅 | 推荐春熙路/太古里附近更有识别度的正餐火锅门店 |
| 太古里附近逛街喝咖啡 | 生成“商场 -> 咖啡/饮品”的顺序候选路线 |
| 九眼桥晚上喝酒，顺便找点夜宵 | 匹配夜间时段，生成“酒吧 -> 夜宵”的组合方案 |
| 天府广场附近出差，1小时内找午餐和咖啡 | 优先短距离、高效率、可执行的商务路线 |
| 随便安排一下 | 返回可点击的方向选择，例如餐饮、景点公园、咖啡茶馆、夜间活动 |

## 系统架构

```text
Web 工作台
  -> 应用服务层
    -> 交互理解与会话上下文
    -> 路线规划与排序引擎
      -> 数据适配层
        -> POI / 评分与口碑 / 营业时间 / 路网 / 语义向量
```

| 模块 | 作用 |
|------|------|
| Web 工作台 | 输入目标、展示地图、时间轴、推荐理由和服务范围 |
| 应用服务层 | 规范化请求、识别服务范围、地标重定位、整合会话上下文 |
| 交互理解 | 支持多轮表达、多人对话、偏好和避让信息 |
| 规划引擎 | 负责意图判断、候选筛选、排序、路线构建和质量治理 |
| 数据适配层 | 统一接入 POI、评分、营业时间和路径服务 |

## API 概览

### POST `/api/plan`

请求示例：

```json
{
  "goal": "春熙路附近想吃火锅",
  "session_id": "sample-session",
  "user_id": "sample-user",
  "city": "chengdu",
  "center_lat": 30.65732,
  "center_lng": 104.08099,
  "radius": 3000,
  "user_mode": "tourist"
}
```

响应包含：

- `variants`：路线方案或单点候选。
- `route` / `recommendations`：推荐 POI、时间、移动距离和地图折线。
- `recommendation_basis`：结构化推荐依据。
- `review_summary`：面向用户的口碑摘要。
- `interaction`：会话、澄清、多轮上下文信息。
- `service_area`：当前服务范围说明。
- `performance`：加载与规划耗时。

### 其他接口

| 接口 | 说明 |
|------|------|
| `GET /api/health` | 服务健康状态 |
| `GET /api/ready` | 数据资产和服务就绪状态 |
| `GET /api/session` | 查看会话和轻画像摘要 |
| `POST /api/session/clear` | 清除当前会话记忆 |
| `POST /api/profile/clear` | 清除指定用户的轻画像 |
| `POST /api/feedback` | 写入偏好、避让和饮食反馈 |

## 推荐依据

RouteMind 智行策的推荐排序综合考虑以下信号：

- POI 质量评分。
- 评价热度估计。
- 品牌和门店实体可信度。
- 用户目标中的类型和语义需求。
- 营业时间与预计到达时段。
- 起点距离、移动时间和路线连贯性。
- 同类重复控制。
- 组合路线中的顺序约束。

系统不会生成不存在的门店，也不会用停车、入口、充电、快递、普通销售点等低价值设施凑路线。

## 数据资产

| 数据 | 当前规模 | 用途 |
|------|----------|------|
| POI 基础信息 | 47,045 个 | 地点候选、类型识别、空间过滤 |
| 评分与口碑索引 | 47,045 条 | 质量分、热度估计、推荐依据 |
| 营业时间 | 47,045 条 | 到达时段可行性 |
| 路网数据 | 291,460 条边 | 移动距离、移动时间、路线折线 |
| 语义向量 | 47,045 x 1024 | 复杂目标候选增强 |

当前数据覆盖成都武侯区与锦江区，用于验证产品在真实城市区域中的规划能力。扩展到其他城市时，可按城市接入对应 POI、评分、营业时间和路径服务。

## 用户模式

| 模式 | 适合用户 | 推荐倾向 |
|------|----------|----------|
| 游客 | 城市游览、半日/一日安排 | 景点、公园、特色餐饮、拍照点 |
| 出差 | 午休、会后、客户餐叙 | 短距离、高效率、商务餐饮 |
| 居民 | 日常休闲、周末放松 | 社区周边、茶馆、餐饮、公园 |

## Web 工作台

- 左侧：自然语言输入、动态目标建议、模式、位置、半径、会话控制。
- 中间：地图、起点、路线折线、POI marker、图层控制。
- 右侧：方案卡、路线时间轴、精选口碑、推荐理由、复制路线和 POI 名单。
- 移动端：规划、地图、结果三视图切换。

## 扩展路线

RouteMind 智行策的规划引擎与数据来源解耦，后续可以逐步接入：

- 线上 POI 搜索与门店详情。
- 真实评分、评价热度和营业时间。
- 地图路径规划。
- 城市级缓存和服务网关。
- 更多城市数据分片。

## 服务边界

当前在线体验范围为成都武侯区与锦江区。系统会对暂未覆盖地区给出明确提示。对于数据资产中缺少的点位，RouteMind 智行策不会编造门店或景点，而是返回已有数据支持的候选结果。
