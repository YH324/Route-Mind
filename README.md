# RouteMind 智能路线规划系统

RouteMind 是面向成都本地生活的自然语言路线规划产品。用户可以直接描述目标，例如“春熙路附近想吃火锅”“太古里附近逛街喝咖啡”“九眼桥晚上喝酒顺便找点夜宵”，系统会理解意图、筛选 POI、生成路线或候选推荐，并解释每个推荐点的选择依据。

**项目试用**：[http://124.223.28.124/](http://124.223.28.124/)

当前服务范围为成都武侯区与锦江区。超出覆盖范围的城市或区县会收到明确提示，系统不会返回缺少数据支持的推荐。

## 核心能力

- **自然语言意图理解**：识别单点推荐、两三点组合路线、半日/一日路线、夜间活动和商务短程等不同目标。
- **动态时间预算**：用户未说明时长时，系统按意图形态和活动组合估算合理用时，不把所有请求固定成同一时间方案。
- **多方案输出**：完整路线支持紧凑高效、休闲慢游、美食探店等不同风格。
- **地图路线表达**：路线结果包含时间轴、移动距离、移动时间和地图折线。
- **推荐依据可读**：每个 POI 都包含结构化推荐依据和面向用户的口碑摘要。
- **数据驱动排序**：综合评分、评价热度估计、品牌识别、商圈密度、营业状态、移动成本、实体可信度和用户语义需求。
- **多轮交互**：支持同一会话下沿用上次位置和偏好，例如“那附近换成茶馆”“还有别的吗”。
- **模糊意图澄清**：当用户只说“随便安排一下”时，系统会先询问方向，而不是强行生成低确定性的路线。
- **服务范围透明**：当前只覆盖成都武侯区与锦江区，其他地区会明确提示暂不支持。

## 系统架构

```text
Web 工作台
  -> 应用服务层
    -> 交互理解与会话上下文
    -> 路线规划与排序引擎
      -> POI 数据、评分索引、营业时间、路网距离、语义向量
```

| 层级 | 说明 |
|------|------|
| Web 工作台 | 自然语言输入、服务范围提示、地图路线、方案对比、推荐详情 |
| 应用服务层 | 请求规范化、服务范围判断、地标中心识别、会话上下文整合 |
| 规划引擎 | 意图分类、动态时间预算、候选筛选、实体治理、评分排序、路线构建 |
| 数据适配层 | 统一 POI、评分、路径和营业时间接口，便于接入线上数据服务 |

## 数据资产

| 数据 | 当前规模 | 用途 |
|------|----------|------|
| POI 基础信息 | 47,045 个 | 地点搜索、类型识别、空间过滤 |
| 评分/口碑聚合索引 | 47,045 条 | 质量分、热度估计、推荐依据 |
| 路网数据 | 291,460 条边 | 路线距离、移动时间、地图折线 |
| 营业时间 | 47,045 条 | 到达时段可行性校验 |
| 语义向量 | 47,045 x 1024 | 复杂需求的语义候选增强 |

数据结构按城市分片设计，后续接入更多城市时可以按城市加载对应 POI、路网、营业时间和评分资产。

## 推荐质量治理

RouteMind 会主动处理常见 POI 推荐问题：

- 过滤停业、装修、入口、门岗、停车、快递、充电等低价值点位。
- 火锅优先完整正餐门店，降低麻辣烫、冒菜、甜品、附属设施等错配结果。
- 茶馆、咖啡、公园、商场、超市、酒吧等类型会结合名称、类型编码和实体特征综合判断。
- 组合需求会严格保留用户说出的顺序，例如“酒吧 -> 夜宵”“商场 -> 咖啡”“午餐 -> 咖啡”。
- 严格时间或移动约束下无法形成完整路线时，返回可执行的顺序候选路线，不硬凑无关地点。

## Web 工作台

- 左侧：自然语言输入、动态目标建议、用户模式、中心点、半径、会话和画像清除。
- 中间：Leaflet 地图、路线折线、起点定位、POI marker、图层控制。
- 右侧：方案对比、可滚动路线列表、精选口碑摘要、推荐理由、复制路线和 POI 名单。
- 移动端：规划、地图、结果三种视图，适配窄屏操作。

## 用户模式

| 模式 | 适合场景 | 推荐倾向 |
|------|----------|----------|
| 游客 | 城市游览、半日/一日路线 | 景点、公园、特色餐饮、可拍照点位 |
| 出差 | 午休、会后、客户餐叙 | 移动距离短、效率优先、商务餐饮 |
| 居民 | 日常休闲、周末放松 | 社区周边、茶馆、餐饮、公园、日常便利 |

## 隐私原则

RouteMind 只使用路线规划所需的信息，不收集手机号、真实身份、消费记录或长期位置轨迹。会话记忆可清除，长期轻画像只保存偏好、避让和饮食限制摘要，并支持通过接口删除。

详细说明见 [隐私设计白皮书](PRIVACY.md)。

## 扩展路径

RouteMind 的规划引擎与数据来源解耦。后续可以逐步接入：

- 线上 POI 搜索与门店详情服务。
- 真实评分、评价热度和营业时间服务。
- 地图路径规划服务。
- Redis 城市缓存和高并发服务网关。
- 更多城市数据分片。

详细说明见 [数据服务扩展指南](docs/API_MIGRATION.md)。
