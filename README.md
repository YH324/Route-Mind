# RouteMind 智行策 —— 自然语言本地生活路线规划

**在线体验**：http://124.223.28.124/

RouteMind 智行策是一款面向本地生活场景的自然语言路线规划产品。用户用日常语言描述出行需求，例如"春熙路附近想吃火锅，吃完想逛街"，系统自动理解意图、筛选地点、生成路线并解释推荐理由。

当前在线体验覆盖成都武侯区与锦江区。

---

## 产品演示

| 场景 | 用户输入 | 系统响应 |
|------|---------|---------|
| 单点推荐 | "春熙路附近吃火锅" | 推荐 2-3 家营业中的火锅店，附口碑摘要和营业时间 |
| 两点串联 | "吃完火锅去茶馆" | 生成火锅→茶馆的路线，检查距离、移动时间和绕行 |
| 半日路线 | "太古里附近半日游" | 生成 3-5 个 POI 的完整路线，估算总用时和移动成本 |
| 多轮细化 | 第一轮"春熙路吃火锅" → 第二轮"吃完想逛街" | 自动继承位置，LLM 理解上下文，生成火锅→购物路线 |
| 模糊意图 | "附近有什么好玩的" | 提示澄清（餐饮/景点/咖啡茶馆/夜间活动），降低选择成本 |

**多轮对话示意**：

```
用户：春熙路吃火锅
系统：推荐 3 家火锅店（附地图、口碑、营业时间）

用户：吃完想逛街
系统：理解为先吃火锅再购物，生成火锅→商场路线

用户：换一个便宜的
系统：替换为性价比更高的组合，保持位置上下文
```

## 核心能力

### 1. 自然语言意图理解
支持单点推荐、两点/三点组合、半日/一日路线、夜间活动、商务短程和模糊目标。大模型参与意图分类和上下文理解，确定性语义门控保护高置信场景，避免短查询被过度扩展。

### 2. 多轮对话
同一会话内自动继承位置、类型偏好和已推荐结果。用户可逐轮细化需求，无需在每条消息中重复地点和约束。

### 3. 动态时间预算
用户未说明时长时，系统根据目标类型和活动组合自动估算合理用时。火锅约 1.5 小时，景点约 2 小时，咖啡约 1 小时——不同组合生成不同总时长，而非固定套用同一套方案。

### 4. 数据驱动推荐
综合质量评分、口碑热度、品牌识别、商圈密度、营业状态、语义匹配度和移动成本进行排序。实体治理层过滤入口、停车、快递、充电、停业装修等低价值点位。

### 5. 路线可行性
不仅考虑地点质量，还检查起点到 POI、POI 之间的移动时间、距离、绕行比例和总移动预算。路线附带真实道路级折线，可在地图上直接查看。

### 6. 可解释结果
每张推荐卡片展示：推荐理由、精选口碑摘要、当前营业状态、与上一站的移动距离和时间。用户知道"为什么推荐这家"。

## 技术架构

```
Web 工作台
  → 应用服务层（请求规范化、服务范围判断、会话上下文整合）
  → 交互理解层（多轮对话追踪、需求推断、意图澄清）
  → 路线规划引擎（意图分类、候选召回、排序、路线构建）
  → 数据适配层（POI / 评分口碑 / 营业时间 / 路网 / 语义向量）
```

**关键技术点**：
- 规划引擎不依赖固定门店清单或路线模板，每次请求从零构建
- LLM 仅参与意图理解和候选评审，不直接生成不存在的 POI
- 多城市扩展设计：数据接口统一抽象，接入新城市的 POI、评分、营业时间和路径服务后即可提供同类能力
