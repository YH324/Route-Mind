# RouteMind 智行策

RouteMind 智行策是面向本地生活和城市出行的自然语言路线规划产品。用户可以直接描述目标，例如“春熙路附近想吃火锅”“太古里附近逛街喝咖啡”“九眼桥晚上喝酒顺便找点夜宵”，系统会理解意图、筛选 POI、生成路线或候选推荐，并解释每个推荐点的选择依据。

**项目试用**：[http://124.223.28.124/](http://124.223.28.124/)

当前在线体验覆盖成都武侯区与锦江区。RouteMind 的产品和数据接口按多城市扩展设计；接入其他城市的 POI、评分口碑、营业时间和路径服务后，可以提供同类规划能力。超出当前覆盖范围的请求会收到明确提示，系统不会用缺少数据支持的地点替代推荐。

## 核心能力

- 自然语言意图理解：识别单点推荐、两三点组合路线、半日/一日路线、夜间活动、商务短程和模糊目标。
- 动态时间预算：用户未说明时长时，根据目标类型和活动组合估算合理用时，不把所有请求固定成同一套时间方案。
- 数据驱动推荐：综合质量评分、口碑热度估计、品牌识别、商圈密度、营业状态、实体可信度、语义需求和移动成本。
- 路线可行性检查：规划不仅考虑地点，还检查起点到 POI、POI 之间的移动时间、距离、绕行比例和总移动预算。
- 组合意图保真：用户明确说出的类型优先覆盖；“顺便、同时”这类弱顺序表达可在减少绕行时调整访问顺序，并在结果中标记。
- 可解释结果：推荐卡片和地图弹窗展示面向用户的推荐理由、精选口碑摘要、营业状态、移动距离和路线折线。
- 多轮交互：支持沿用同一会话中的位置、目标和偏好，也支持模糊意图澄清。

## 系统架构

```text
Web 工作台
  -> 应用服务层
  -> 交互理解与会话上下文
  -> 路线规划与排序引擎
  -> 数据适配层
  -> POI / 评分口碑 / 营业时间 / 路网 / 语义向量
```

| 层级 | 说明 |
|------|------|
| Web 工作台 | 自然语言输入、服务范围提示、地图路线、方案对比、推荐理由展示 |
| 应用服务层 | 请求规范化、服务范围判断、地标中心识别、会话上下文整合 |
| 规划引擎 | 意图分类、动态时间预算、候选筛选、实体治理、排序、路线构建 |
| 数据适配层 | 统一 POI、评分、营业时间和路径接口，便于替换为线上数据服务 |

## 后端模型链路

RouteMind 不依赖固定门店清单或固定路线模板。一次请求会经过：

1. 意图识别：大模型可用时用于理解模糊表达；确定性语义门控保护高置信场景，避免把短查询误扩成路线。
2. 候选召回：按服务范围、起点、半径和显式目标召回 POI，并对用户明确说出的类型做保底保留。
3. 实体治理：修正火锅、茶馆、咖啡、公园、商场、超市、酒吧等高频类型的误分类，过滤入口、停车、快递、充电、停业装修等低价值点位。
4. 排序打分：综合质量、热度、实体可信度、品牌/商圈信号、营业状态、语义需求和移动成本。
5. 路线生成：使用路网或距离图计算移动时间，检查单段距离、总移动时间和绕行比例。
6. 路线 slate 评审：对候选路线做 listwise 比较；外部大模型可用时可以参与路线评审，但不会编造新地点。

## 大模型配置

后端支持 OpenAI-compatible Chat Completions 接口。当前 provider 优先级为 DeepSeek -> MiMo -> MiniMax -> GLM -> 本地语义兜底。启用 DeepSeek 时，在环境变量或 `.env` 中配置：

```text
DEEPSEEK_API_KEY=your_api_key
DEEPSEEK_CHAT_URL=https://api.deepseek.com/chat/completions
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_AUTH_TYPE=bearer
```

大模型只参与意图理解和已召回候选/路线的评审，不直接生成不存在的 POI。
